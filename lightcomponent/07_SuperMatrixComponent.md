# 07 - 矩阵灯组件 (SuperMatrixComponent)

> **所属模块**: SuperStage 运行时  
> **适用对象**: 灯光设计师、灯具蓝图制作者  
> **前置阅读**: [00 - 光源组件总览](00_LightComponent_Overview.md)

---

## 一、概述

**SuperMatrixComponent** 是 SuperStage 中用于实现**矩阵像素级控制**的光源组件。它将一个灯具面板划分为多个独立的「分段」（Segment），每个分段可以独立控制颜色，从而实现像素阵列、LED 灯条、矩阵灯等效果。

除了材质层面的分段着色，SuperMatrixComponent 还支持为每个分段创建**真实光源**（PointLight 或 SpotLight），让每个像素不仅在视觉上发光，还能真正照亮周围环境。

### 适用场景

- **矩阵灯** — 如 Robe Robin 600 LEDWash 等矩阵控制灯具
- **LED 灯条** — 可寻址 LED 灯带，每个像素独立控制
- **像素面板** — LED 像素屏或装饰面板
- **多段 Wash 灯** — 多区域独立调色的 Wash 灯具

---

## 二、组件结构

```
SuperMatrixComponent (继承自 USceneComponent)
  ├── YStaticMeshMatrix  — 矩阵面板模型（静态网格，承载分段材质）
  └── 分段光源[]          — 每个分段的真实光源（可选）
        ├── SegmentPointLights[]  — PointLight 数组（球形发光）
        └── SegmentSpotLights[]   — SpotLight 数组（定向发光）
```

### 材质系统

| 材质 | 说明 |
|------|------|
| **不透明材质 (MatrixMaterial)** | 标准矩阵分段材质，不透明渲染 |
| **透明材质 (MatrixMaterialTransparent)** | 半透明矩阵分段材质 |

通过 `bTransparent` 开关切换两种模式。材质内部通过分段参数（SegCount、颜色数组）实现像素级着色。

---

## 三、默认参数 (DefaultParameter)

### 3.1 分段配置

| 参数 | 说明 | 范围 | 默认值 |
|------|------|------|--------|
| **SegCount** | 分段数量（像素数） | 1 ~ 200 | 5 |
| **bUseUAxis** | 分段方向轴 | 布尔 | true |

#### 分段方向说明

| bUseUAxis | 分段方向 | 说明 |
|-----------|----------|------|
| **true**（默认） | 沿 U 轴（X 轴）分段 | 像素从左到右排列 |
| **false** | 沿 V 轴（Y 轴）分段 | 像素从上到下排列 |

```
bUseUAxis = true（沿X轴分段）:
┌───┬───┬───┬───┬───┐
│ 0 │ 1 │ 2 │ 3 │ 4 │  ← 5个分段，从左到右
└───┴───┴───┴───┴───┘

bUseUAxis = false（沿Y轴分段）:
┌─────────┐
│    0    │
├─────────┤
│    1    │
├─────────┤  ← 5个分段，从上到下
│    2    │
├─────────┤
│    3    │
├─────────┤
│    4    │
└─────────┘
```

> **提示**：分段方向应与灯具的实际像素排列方向一致。对于水平排列的 LED 灯条使用 U 轴，对于垂直排列的 LED 条使用 V 轴。

### 3.2 面板模型

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| **StaticMeshMatrix** | 矩阵面板使用的静态网格模型 | 静态网格引用 | 无 |
| **MatrixTransform** | 面板相对于组件的位置、旋转和缩放偏移 | 变换 | 无偏移 |

### 3.3 材质配置

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| **bTransparent** | 是否使用透明材质 | 布尔 | false |

### 3.4 组件级亮度分控

| 参数 | 说明 | 范围 | 默认值 |
|------|------|------|--------|
| **ComponentDimmer** | 组件级亮度分控 | 0.0 ~ 1.0 | 1.0 |

## 四、分段光源系统

### 4.1 光源类型选择

| 参数 | 说明 | 类型 |
|------|------|------|
| **SegmentLightType** | 每个分段创建的真实光源类型 | 枚举 |

| 枚举值 | 说明 | 适用场景 |
|--------|------|----------|
| **None** | 不创建真实光源，仅材质发光 | 纯视觉效果，最佳性能 |
| **PointLight** | 每个分段创建一个点光源 | 需要全方位照明的矩阵灯 |
| **SpotLight** | 每个分段创建一个聚光灯 | 需要定向照明的矩阵灯 |

> **性能影响**：每个分段都会创建一个独立的光源组件。200 个分段 = 200 个光源，这可能会严重影响性能。请根据实际需要选择光源数量。

### 4.2 光源参数

| 参数 | 说明 | 范围 | 默认值 |
|------|------|------|--------|
| **SegmentLightIntensity** | 分段光源的基础亮度 | ≥ 0 | 1000.0 |
| **SegmentLightRadius** | 分段光源的衰减半径 | ≥ 1 | 2400.0 cm |
| **SpotOuterConeAngle** | 分段 SpotLight 的外锥角（仅 SpotLight 类型有效） | 1 ~ 90 | 90.0 度 |

#### 分段光源亮度计算

每个分段光源的最终亮度由以下因素共同决定：

```
最终亮度 = SegmentLightIntensity × 颜色亮度 × 颜色Alpha × 整体亮度 × ComponentDimmer × 频闪倍率
```

其中：
- **颜色亮度** = 像素颜色的感知亮度（RGB 加权平均）
- **颜色Alpha** = 颜色的 Alpha 通道（0 = 未启用，1 = 已启用）

> **设计原理**：暗色像素产生的光照弱，亮色像素产生的光照强，与现实中 LED 的表现一致。

### 4.3 光源位置

分段光源的位置根据以下逻辑自动计算：

1. 获取矩阵面板模型的包围盒
2. 根据分段方向（U 轴或 V 轴）计算每个分段的中心位置
3. 将位置变换到组件本地坐标系

```
分段中心位置 = 网格包围盒最小点 + 包围盒尺寸 × ((分段索引 + 0.5) / 总分段数)
```

每个光源附加在 SuperMatrixComponent 上，跟随灯具运动。

### 4.4 光源属性

所有分段光源共享以下属性：

| 属性 | 值 | 说明 |
|------|-----|------|
| 移动性 | 可移动 | 支持实时变化 |
| 投射阴影 | 关闭 | 避免大量光源的阴影开销 |
| 初始亮度 | 0 | 由 DMX 数据驱动 |
| 初始颜色 | 黑色 | 由 DMX 数据驱动 |

> **SpotLight 特殊设置**：  
> - 内锥角固定为 1°（硬核心）  
> - 外锥角由 SpotOuterConeAngle 参数控制  
> - 朝下方照射（旋转 90° 向下）

## 五、DMX 可控参数

### 5.1 分段颜色

| 操作 | 说明 |
|------|------|
| **设置分段颜色** | 为指定索引的分段设置 RGB 颜色 |

| 参数 | 说明 | 范围 |
|------|------|------|
| **Index** | 分段索引（从 0 开始） | 0 ~ (SegCount - 1) |
| **RGB** | 分段颜色 | RGB 线性颜色 |

设置分段颜色时会自动：
1. 将颜色 Alpha 设为 1.0（标记该分段已启用）
2. 更新材质参数（分段着色）
3. 同步更新对应的分段光源颜色和亮度

### 5.2 整体亮度 (Intensity)

| 操作 | 说明 | 参数范围 |
|------|------|----------|
| **设置亮度** | 控制所有分段和光源的整体亮度 | 0.0（全灭）~ 1.0（全亮） |

亮度通过材质参数 `Brightness` 控制，同时影响所有分段光源的强度。

### 5.3 频闪 (Strobe)

矩阵组件拥有**独立的频闪系统**：

| 参数 | 说明 | 范围 |
|------|------|------|
| **频闪速度** | 闪烁的快慢 | ≥ 0 |
| **频闪模式** | 闪烁的波形（0~7） | 与基类一致 |
| **随机种子** | 随机模式的种子值 | 任意 |

频闪模式列表：

| 模式 | 名称 | 效果 |
|------|------|------|
| 0 | Closed | 完全关闭 |
| 1 | Open | 常亮（默认） |
| 2 | Linear | 渐亮渐灭 |
| 3 | Pulse | 方波闪烁 |
| 4 | Ramp Up | 渐亮 |
| 5 | Ramp Down | 渐灭 |
| 6 | Sine | 正弦脉动 |
| 7 | Random | 随机乱闪 |

> **注意**：频闪会**同时影响**材质亮度和分段光源亮度，确保视觉效果一致。

### 5.4 组件级亮度分控

| 操作 | 说明 | 参数范围 |
|------|------|----------|
| **设置 ComponentDimmer** | 独立控制该矩阵组件的亮度 | 0.0 ~ 1.0 |

## 六、使用指南

### 6.1 基本设置流程

1. 在灯具蓝图中添加 SuperMatrixComponent
2. 设置**静态网格模型**（StaticMeshMatrix）— 通常是一个平面
3. 设置**面板位置和大小**（MatrixTransform）
4. 配置**分段数量**（SegCount）— 与灯具实际像素数匹配
5. 选择**分段方向**（bUseUAxis）— 与灯具像素排列方向匹配
6. 选择**分段光源类型**（SegmentLightType）— 根据是否需要真实光照
7. 调整**光源参数**（亮度、半径、锥角等）
8. 在灯具库中映射 DMX 通道

### 6.2 分段数与性能的权衡

| 分段数 | 材质开销 | 光源开销（启用时） | 适用场景 |
|--------|----------|-------------------|----------|
| 1~10 | 极低 | 低 | 少量分区的 Wash 灯 |
| 10~50 | 低 | 中等 | 标准矩阵灯 |
| 50~100 | 中等 | 较高 | 高密度像素灯 |
| 100~200 | 较高 | 高（谨慎使用） | 超高密度 LED 条 |

> **建议**：如果不需要分段光源照亮环境，将 SegmentLightType 设为 None 可以大幅降低性能开销。

### 6.3 初始化行为

矩阵组件初始化时会执行以下操作：

1. 创建静态网格组件并设置模型
2. 创建动态材质实例（根据透明模式选择材质）
3. 推送核心参数（分段数、分段方向）到材质
4. 将所有分段颜色初始化为**黑色**（避免旧状态残留）
5. 创建分段光源（如果 SegmentLightType 不为 None）
6. 所有光源初始亮度设为 0（由 DMX 数据驱动）

### 6.4 光源重建

以下情况会触发分段光源的**重建**（销毁旧光源并创建新光源）：

- 修改 SegmentLightType（光源类型变更）
- 修改 SegCount（分段数量变更）
- 组件重新注册

> **注意**：光源重建是一个相对较重的操作（涉及组件创建和注册），不应在运行时频繁触发。分段数和光源类型应在灯具蓝图中预设好。

## 七、常见问题

### Q: 分段颜色设置后没有变化？
**A**: 检查以下几点：
1. SegCount 是否设置正确（分段索引不能超出范围）
2. 材质是否正确创建（StaticMeshMatrix 需要有效引用）
3. 整体亮度（Intensity）是否大于 0
4. ComponentDimmer 是否大于 0

### Q: 分段光源创建失败或数量不对？
**A**: 分段光源在以下时机创建：
1. 组件 OnRegister 时
2. 初始化 SetMatrixMaterial 时（仅当光源尚未创建）

确认 SegmentLightType 不为 None，且 SegCount 值正确。

### Q: 如何实现二维矩阵（如 5×5 网格）？
**A**: SuperMatrixComponent 目前仅支持**一维分段**（沿 U 或 V 轴）。要实现二维矩阵，可以使用多个 SuperMatrixComponent，每行/列使用一个组件，或者将 25 个像素展开为 1 行（SegCount=25）并在材质中通过 UV 映射实现二维布局。

### Q: 分段光源的阴影可以开启吗？
**A**: 分段光源默认关闭阴影以保证性能。不建议在大量分段时开启阴影（200 个光源的阴影计算会严重影响帧率）。

### Q: 修改光源半径或锥角需要重建吗？
**A**: 不需要。修改 SegmentLightRadius 和 SpotOuterConeAngle 后，调用 UpdateLightParameters 即可直接更新所有现有光源的参数，无需重建。

### Q: 为什么像素颜色是黑色但光源仍有微弱亮度？
**A**: 黑色像素的亮度计算为 0（颜色亮度 = 0），所以光源亮度应该也是 0。如果仍有微弱亮度，可能是 DMX 数据中颜色值不完全为零，或者存在浮点精度问题。
