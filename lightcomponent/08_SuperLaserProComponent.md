# 08 - 激光 Pro 组件 (SuperLaserProComponent)

> **所属模块**: SuperStage 运行时  
> **适用对象**: 灯光设计师、激光操作员、灯具蓝图制作者  
> **前置阅读**: [00 - 光源组件总览](00_LightComponent_Overview.md)

---

## 一、概述

**SuperLaserProComponent** 是 SuperStage 中用于渲染**激光线效果**的专用组件。它使用程序化网格（Procedural Mesh）技术，根据外部激光控制器（如 Pangolin Beyond）提供的**点数据**，实时生成可见的激光线条几何体。

与传统的粒子系统或 Beam 组件不同，SuperLaserProComponent 的激光线条是基于**精确的数学描述**生成的——每条激光线由起点和终点定义，支持多条线同时渲染，并且可以进行碰撞遮挡检测，在碰到障碍物时自动截断。

### 适用场景

- **激光灯** — 由 Beyond 等激光控制器驱动的激光灯具
- **激光秀** — 多条彩色激光线组成的空间图案
- **激光安全线** — 模拟激光栅栏或安全边界

---

## 二、组件结构

```
SuperLaserProComponent (继承自 USceneComponent)
  └── ProceduralMeshComp  — 程序化网格组件（实时生成激光线几何体）
```

### 渲染方式

激光线通过以下流程渲染：

```
点数据输入 → 变化检测 → 碰撞检测(可选) → 几何体生成 → 材质参数更新 → 渲染
```

每条激光线被构建为一个**四边形条带**（两个三角形组成的矩形面片），面向相机方向，确保从任何视角看都呈现为一条线。

## 三、默认参数 (DefaultParameter)

### 3.1 激光线外观参数

| 参数 | 说明 | 范围 | 默认值 | 单位 |
|------|------|------|--------|------|
| **BeamLength** | 激光线的最大长度（无碰撞遮挡时的默认长度） | 100 ~ 50000 | 5000.0 | cm |
| **ProjectionAngle** | 投射角度范围，控制点数据 X/Y [-1,1] 映射的锥角 | 1 ~ 90 | 30.0 | 度 |
| **LaserWidth** | 激光线的宽度（粗细） | 0.1 ~ 20 | 1.0 | cm |
| **CoreSharpness** | 激光线核心的锐利程度，值越高核心越集中 | 1 ~ 10 | 2.0 | — |
| **DepthFade** | 深度衰减，控制激光线远端变暗的程度。0=不衰减，1=完全衰减 | 0 ~ 1 | 0.5 | — |

#### 参数说明

- **BeamLength**：当没有碰撞遮挡时，激光线从起点延伸到这个距离。如果开启了碰撞检测且碰到障碍物，实际长度会更短。

- **LaserWidth**：控制激光线的视觉粗细。真实激光非常细（约 1mm），但在虚拟场景中为了可见性通常设置得更宽。

- **CoreSharpness**：控制激光线核心（最亮部分）的集中程度。高值使核心更细锐，低值使核心更柔和扩散。

- **DepthFade**：当激光线穿过或触碰到物体时，这个参数控制过渡区域的长度。大值使过渡更柔和，小值使过渡更生硬。

### 3.2 亮度与透明度参数

| 参数 | 说明 | 范围 | 默认值 |
|------|------|------|--------|
| **Dim** | 自发光强度，指数曲线亮度（1=正常，3=较亮，10=最亮） | 1.0 ~ 10.0 | 10.0 |
| **OpacityScale** | 激光线的不透明度缩放，0=完全透明，1=完全不透明 | 0.0 ~ 1.0 | 0.1 |
| **SpotDimmer** | 激光光斑亮度，打在物体表面时的光斑效果 | 0.0 ~ 5.0 | 1.0 |
| **ComponentDimmer** | 组件级亮度分控，与 Dim 乘法叠加：FinalDim = Dim × ComponentDimmer | 0.0 ~ 1.0 | 1.0 |

### 3.3 烟雾效果参数

| 参数 | 说明 | 范围 | 默认值 |
|------|------|------|--------|
| **FogSpeed** | 烟雾纹理在激光线上流动的速度，0=静止，2=快速流动 | 0 ~ 2 | 0.3 |
| **FogInfluence** | 烟雾纹理对激光线的影响程度（0=无烟雾，1=完全受烟雾控制） | 0 ~ 1 | 0.5 |

> **使用场景**：当舞台上有 Haze 烟雾时，增加 FogInfluence 可以让激光线呈现更真实的在烟雾中穿行的效果，烟雾纹理会沿着激光线流动。

### 3.4 碰撞检测

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| **bEnableCollision** | 是否启用碰撞遮挡检测 | 布尔 | false |

#### 碰撞检测工作原理

当 `bEnableCollision` 启用时：

1. 对每条激光线从起点向终点方向发射射线
2. 检测射线与场景中物体的碰撞交点
3. 如果碰到障碍物，将激光线截断在碰撞点处
4. 忽略灯具自身 Actor 的碰撞

```
启用碰撞检测:
 灯具 ━━━━━━━━━●━━━━× 墙壁
              碰撞点   (激光在此截断)

未启用碰撞检测:
 灯具 ━━━━━━━━━━━━━━━━━━━━━━━●
                            (延伸到 BeamLength)
```

> **性能提示**：碰撞检测需要对每条激光线执行射线追踪，多条激光线时会有明显开销。不需要时建议关闭。

### 3.5 激光材质

| 参数 | 说明 | 类型 |
|------|------|------|
| **BeamMaterial** | 激光线使用的材质 | 材质引用 |

> 默认使用 SuperStage 提供的激光材质（基于 SuperLaserPro.usf 自定义着色器），支持核心锐利度、深度渐隐、烟雾流动等效果。

## 四、DMX 可控参数

### 4.1 激光点数据 (Laser Points)

激光组件的核心输入是**点数据数组**。每个点包含以下信息：

| 属性 | 说明 |
|------|------|
| **位置** | 点在空间中的位置 |
| **颜色** | 该点的 RGB 颜色 |

点数据通常由外部激光控制器（如 Pangolin Beyond）通过网络协议发送，SuperStage 的激光系统负责接收并传递给激光组件。

#### 变化检测

激光组件内置了**变化检测**机制：

- 比较新旧点数据的数量和位置
- 只有当点数据**实际发生变化**时才重建网格
- 避免在点数据不变时进行不必要的几何体重建

### 4.2 可见性控制

| 操作 | 说明 |
|------|------|
| **设置激光可见性** | 控制整个激光组件的显示/隐藏 |

当可见性关闭时，程序化网格不渲染，碰撞检测也会跳过。

### 4.3 组件级亮度分控

| 操作 | 说明 | 参数范围 |
|------|------|----------|
| **设置 ComponentDimmer** | 控制该激光组件的整体亮度 | 0.0 ~ 1.0 |

ComponentDimmer 会影响所有激光线的材质亮度参数。

## 五、几何体生成

### 5.1 网格构建过程

激光组件收到点数据后，执行以下步骤生成可见的激光线：

1. **解析点数据** — 提取每条激光线的起点、终点和颜色
2. **碰撞检测**（如果启用）— 对每条线进行射线追踪，截断被遮挡的线段
3. **生成顶点** — 每条激光线生成 4 个顶点（两端各 2 个，组成矩形条带）
4. **计算法线** — 法线指向相机方向，确保条带始终面向观察者
5. **生成 UV** — 用于材质的纹理映射
6. **设置顶点颜色** — 每个顶点携带激光线的颜色信息
7. **创建网格段** — 使用程序化网格组件渲染

### 5.2 多条激光线

一个激光组件可以同时渲染**多条**激光线。每条线独立计算碰撞和颜色。所有线条合并在一个程序化网格中渲染，效率较高。

## 七、使用指南

### 7.1 基本设置流程

1. 在灯具蓝图中添加 SuperLaserProComponent
2. 配置**激光线外观**参数（宽度、锐利度、深度渐隐等）
3. 配置**激光线长度**（BeamLength）
4. 决定是否开启**碰撞检测**（bEnableCollision）
5. 配置**烟雾效果**（FogSpeed、FogInfluence）
6. 连接激光控制器数据源（如 Beyond）

### 7.2 视觉效果调节

#### 锐利 vs 柔和

| 效果 | CoreSharpness | LaserWidth | 说明 |
|------|---------------|------------|------|
| 锐利激光 | 高（10+） | 小（0.3） | 细而锐利，接近真实激光 |
| 柔和激光 | 低（1~3） | 大（1.0+） | 宽而柔和，更具视觉冲击 |

#### 烟雾浓度

| 效果 | FogInfluence | FogSpeed | 说明 |
|------|-------------|----------|------|
| 无烟雾 | 0.0 | — | 干净的激光线 |
| 轻烟 | 0.3 | 0.5 | 微妙的烟雾流动 |
| 浓烟 | 0.8+ | 1.0+ | 明显的烟雾散射 |

#### 亮度层级

激光组件有多层亮度控制：

```
最终亮度 = Dim × ComponentDimmer
最终光斑 = SpotDimmer × ComponentDimmer
最终透明度 = OpacityScale
```

- **Actor 级 Dimmer** — 整台灯具的主亮度
- **ComponentDimmer** — 该激光组件的亮度分控
- **Dim** — 材质层面的亮度细调
- **OpacityScale** — 不透明度缩放

### 7.3 碰撞检测配置

#### 何时启用碰撞检测

| 场景 | 建议 |
|------|------|
| 激光照射地面/墙壁 | 启用 — 激光在接触面截断，更真实 |
| 激光在空中扫射 | 关闭 — 无需碰撞，节省性能 |
| 大量激光线 | 关闭 — 避免大量射线追踪的开销 |
| 交互式激光 | 启用 — 激光与移动物体交互 |

## 八、常见问题

### Q: 激光线没有显示？
**A**: 检查以下几点：
1. 是否有有效的**点数据**输入（至少需要 2 个点定义一条线）
2. **BeamLength** 是否足够大
3. **可见性**是否开启
4. **ComponentDimmer** 和 **Dim** 是否大于 0
5. **BeamMaterial** 是否有效

### Q: 激光线在移动时有明显的闪烁/抖动？
**A**: 激光组件内置了变化检测，只有点数据变化时才重建网格。如果闪烁是由材质导致的，尝试调整 FogSpeed 或 DepthFade 参数。

### Q: 碰撞检测不准确？
**A**: 碰撞检测使用标准的线性射线追踪（Line Trace），需要确保目标物体有正确的碰撞体设置。使用复杂碰撞（Complex Collision）可以获得更精确的结果。

### Q: 如何控制单条激光线的颜色？
**A**: 每条激光线的颜色由**点数据**中的颜色属性决定。在激光控制器（如 Beyond）中设置每条线的颜色即可。颜色信息通过顶点颜色传递到材质。

### Q: 激光组件会产生真实光照吗？
**A**: 不会。激光线通过自发光材质渲染，不产生真实光照。如果需要激光照射到物体上产生光点效果，需要额外的逻辑（如在碰撞点放置光源或使用贴花）。

### Q: 大量激光线的性能表现如何？
**A**: 所有激光线合并在一个程序化网格中渲染，Draw Call 数量为 1，几何体生成在 CPU 端完成。主要性能瓶颈是：
- **碰撞检测** — 每条线一次射线追踪（启用时）
- **网格重建** — 点数据变化时的顶点/三角形/法线计算
- **材质渲染** — 半透明材质的 GPU 开销
