# 09 - 升降控制组件 (SuperLiftComponent)

> **所属模块**: SuperStage 运行时  
> **适用对象**: 灯光设计师、灯具蓝图制作者  
> **前置阅读**: [00 - 光源组件总览](00_LightComponent_Overview.md)

---

## 一、概述

**SuperLiftComponent** 是 SuperStage 中最简单的光源辅助组件，用于控制灯具沿 **Z 轴（垂直方向）** 的升降运动。它模拟真实舞台中灯具挂点的电动升降机构，允许通过 DMX 通道实时调整灯具的悬挂高度。

与其他光源组件不同，升降组件**不涉及任何光照或材质渲染**——它纯粹是一个运动控制组件，通过改变组件的相对位置（Z 轴偏移）来实现升降效果。

### 适用场景

- **可升降灯具** — 悬挂在电动吊杆上的灯具，需要实时调整高度
- **升降效果** — 灯具在演出中的上升/下降动作
- **动态舞台** — 需要灯具高度随场景变化的应用

---

## 二、组件结构

```
SuperLiftComponent (继承自 USceneComponent)
  └── (无子组件)
```

> **最简组件**：升降组件不创建任何子组件，不加载任何材质，不启用 Tick——它仅在接收到升降指令时更新自身的相对位置。

---

## 三、工作原理

### 3.1 升降运动模型

升降组件的运动基于以下模型：

```
初始位置 (Z=0)                    最大升降位置 (Z=LiftRange)
     │                                    │
     ▼                                    ▼
     ┃════════════════════════════════════┃
     0.0           DMX值             1.0
```

DMX 通道值（0~1）会被映射到灯具的升降范围内：
- **DMX = 0.0** → 灯具在最低位置（Z = 0）
- **DMX = 1.0** → 灯具在最高位置（Z = LiftRange）

### 3.2 平滑插值

升降运动使用**平滑插值**（Interpolation），避免灯具瞬间跳到目标位置：

```
当前位置 → 平滑过渡 → 目标位置
```

插值计算：
```
新Z位置 = 当前Z位置 + (目标Z位置 - 当前Z位置) × 插值速度 × DeltaTime
```

其中：
- **目标Z位置** = DMX值 × 升降范围（LiftRange）
- **插值速度** = 灯具设定的升降速度（LiftSpeed）
- **DeltaTime** = 帧间隔时间

> **效果**：灯具不会瞬间跳到目标高度，而是以设定的速度平滑过渡，模拟真实电机的运动特性。

### 3.3 值域钳制

DMX 输入值会被自动钳制到 [0, 1] 范围：
- 小于 0 的值会被限制为 0
- 大于 1 的值会被限制为 1

## 四、DMX 可控参数

### 4.1 升降位置 (Lift Z)

| 参数 | 说明 | 范围 |
|------|------|------|
| **LiftZ** | 归一化的升降位置 | 0.0（最低）~ 1.0（最高） |

这是升降组件的**唯一控制参数**。

| 输入值 | 行为 |
|--------|------|
| 0.0 | 灯具下降到最低位置 |
| 0.5 | 灯具位于升降范围的中间高度 |
| 1.0 | 灯具上升到最高位置 |

### 4.2 关联参数（由灯具 Actor 提供）

以下参数不在升降组件内部定义，而是由灯具 Actor 在调用升降组件时传入：

| 参数 | 说明 | 来源 |
|------|------|------|
| **LiftRange** | 升降范围（Z 轴最大位移量） | 灯具 Actor 的灯具库定义 |
| **LiftSpeed** | 升降速度（插值速度系数） | 灯具 Actor 的灯具库定义 |
| **DeltaTime** | 帧间隔时间 | 引擎自动提供 |

## 五、使用指南

### 5.1 基本设置

1. 在灯具蓝图中添加 SuperLiftComponent
2. 将升降组件附加在需要升降的组件层级上
3. 在灯具库中配置升降通道（映射到 LiftZ 参数）
4. 设置升降范围（LiftRange）和升降速度（LiftSpeed）

### 5.2 组件层级位置

升降组件应放在灯具组件层级中**需要升降的部分之上**：

```
灯具 Actor
  └── 安装件（固定不动）
        └── SuperLiftComponent ← 升降组件
              └── 灯体（跟随升降运动）
                    ├── 摇臂
                    ├── 灯头
                    └── 光源组件
```

这样，升降组件的 Z 轴位移会传递给所有子组件，实现整个灯体的升降。

### 5.3 性能特点

| 特性 | 说明 |
|------|------|
| **无 Tick** | 组件不启用 Tick，不消耗每帧 CPU 资源 |
| **按需更新** | 仅在接收到 DMX 升降指令时计算位置 |
| **零渲染开销** | 不涉及任何材质或光照，纯位置变换 |

> **注意**：虽然组件本身不 Tick，但它的位置更新依赖外部调用（由灯具 Actor 在其 Tick 中调用 SetLiftZ）。这意味着升降的平滑度取决于灯具 Actor 的更新频率。

## 六、常见问题

### Q: 灯具升降时有明显的跳跃/不平滑？
**A**: 检查以下几点：
1. **LiftSpeed** 是否设置得太大（值越大插值越快，过大可能导致不平滑）
2. 灯具 Actor 的 Tick 频率是否足够（低帧率下插值步长大，可能不够平滑）
3. DMX 数据的更新频率是否稳定

### Q: 灯具升降范围不对？
**A**: LiftRange 参数由灯具 Actor 传入。检查灯具库中的升降范围配置是否正确。升降组件只是将 DMX 值（0~1）乘以 LiftRange 来计算目标位置。

### Q: 灯具无法升降到指定高度？
**A**: 确认 DMX 值已正确映射到 0~1 范围。升降组件会将输入值钳制到 [0, 1]，超出范围的值不会有额外效果。

### Q: 如何实现灯具的急停？
**A**: 将 LiftSpeed 设为非常大的值，灯具会几乎瞬间到达目标位置。但不建议这样做——真实的舞台升降机构有物理限制，急停可能不符合实际效果。

### Q: 升降组件可以控制水平方向的运动吗？
**A**: 不可以。升降组件**仅控制 Z 轴**（垂直方向）。如果需要水平运动，请使用灯具 Actor 的 Pan/Tilt 系统或自定义运动组件。

### Q: 多个升降组件可以串联使用吗？
**A**: 理论上可以，但不推荐。将多个升降组件串联会导致位移叠加，控制逻辑变得复杂。建议一个灯具使用一个升降组件。
