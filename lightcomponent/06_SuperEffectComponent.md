# 06 - 效果平面组件 (SuperEffectComponent)

> **所属模块**: SuperStage 运行时  
> **适用对象**: 灯光设计师、灯具蓝图制作者  
> **前置阅读**: [00 - 光源组件总览](/docs/light-components/overview)

---

## 一、概述

**SuperEffectComponent** 是一个基于**静态网格 + 材质驱动**的视觉效果组件，用于模拟灯具上的 LED 效果面板、频闪面板、装饰灯带等非投射型发光元件。它不会产生真实的光照（不含 SpotLight 或 RectLight），而是通过动态材质的自发光（Emissive）来呈现视觉效果。

### 适用场景

- **LED 效果面板** — 灯具正面或侧面的 LED 装饰面板
- **频闪面板** — 高功率频闪灯的发光面
- **光带效果** — LED 灯条的流水、追逐等动态效果
- **装饰发光** — 灯具上的辅助装饰发光元件

### 与其他光源组件的区别

| 特性 | SuperEffectComponent | SuperSpotComponent / SuperBeamComponent |
|------|---------------------|----------------------------------------|
| **光照投射** | 无（纯视觉效果） | 有（真实光照） |
| **渲染方式** | 静态网格 + 自发光材质 | SpotLight + 光照函数 |
| **效果动画** | 支持效果图案、速度、方向等 | 不支持效果动画 |
| **性能开销** | 较低（仅材质更新） | 较高（光照+阴影计算） |
| **透明模式** | 支持不透明/透明切换 | 不支持 |

---

## 二、组件结构

```
SuperEffectComponent (继承自 USceneComponent)
  └── YStaticMeshEffect  — 效果面板模型（静态网格，承载自发光材质）
```

### 材质系统

效果组件支持两种材质模式：

| 材质 | 说明 |
|------|------|
| **不透明材质 (EffectMaterial)** | 标准自发光材质，不透明渲染，性能较好 |
| **透明材质 (EffectMaterialTransparent)** | 半透明自发光材质，可看到背后物体，适合需要通透感的效果 |

通过 `bTransparent` 开关切换两种模式（见下文）。

---

## 三、默认参数 (DefaultParameter)

### 3.1 效果模型

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| **StaticMeshEffect** | 效果面板使用的静态网格模型 | 静态网格引用 | 无 |
| **EffectTransform** | 效果面板相对于组件的位置、旋转和缩放偏移 | 变换 | 无偏移 |

> **提示**：效果面板模型通常是一个平面（Plane）或自定义形状的面片，UV 展开后可以被效果材质正确着色。

### 3.2 材质配置

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| **EffectMaterialNo** | 材质槽索引，指定动态材质绑定到网格的第几个材质槽 | 整数 | 0 |
| **bTransparent** | 是否使用透明材质 | 布尔 | false |

#### 透明模式说明

| 模式 | 行为 | 适用场景 |
|------|------|----------|
| **不透明** (bTransparent = false) | 效果面板完全遮挡背后物体，渲染性能较好 | 标准 LED 面板 |
| **透明** (bTransparent = true) | 效果面板半透明，可看到背后物体 | 透明灯罩、全息效果 |

> **注意**：透明材质在运行时切换不透明材质会自动重建动态材质实例，确保正确的渲染状态。系统会缓存当前材质的透明状态，避免不必要的重建。

### 3.3 组件级亮度分控

| 参数 | 说明 | 范围 | 默认值 |
|------|------|------|--------|
| **ComponentDimmer** | 组件级亮度分控，与 Actor 级 Dimmer 乘法叠加 | 0.0 ~ 1.0 | 1.0 |

最终亮度公式：
```
最终亮度 = Actor级Dimmer × ComponentDimmer × 频闪倍率
```

## 四、DMX 可控参数

### 4.1 亮度 (Intensity)

| 操作 | 说明 | 参数范围 |
|------|------|----------|
| **设置亮度** | 控制效果面板的自发光亮度 | 0.0（全灭）~ 1.0（全亮） |

亮度通过材质参数 `Brightness` 控制。

### 4.2 颜色 (Color)

| 操作 | 说明 | 参数范围 |
|------|------|----------|
| **设置颜色** | 控制效果面板的发光颜色 | RGB 线性颜色 |

颜色通过材质参数 `LightColor` 控制。

> **示例**：  
> - 白色频闪面板：Color = (1, 1, 1)  
> - 红色效果灯带：Color = (1, 0, 0)  
> - 暖白 LED：Color = (1, 0.85, 0.6)

### 4.3 频闪 (Strobe)

效果组件拥有**独立的频闪系统**，与灯光组件的频闪完全一致：

| 参数 | 说明 | 范围 |
|------|------|------|
| **频闪速度** | 闪烁的快慢 | ≥ 0 |
| **频闪模式** | 闪烁的波形（0~7） | 见下表 |
| **随机种子** | 随机模式的种子值 | 任意 |

#### 频闪模式

| 模式 | 名称 | 效果 |
|------|------|------|
| 0 | Closed | 完全关闭 |
| 1 | Open | 常亮（默认） |
| 2 | Linear | 渐亮渐灭 |
| 3 | Pulse | 方波闪烁 |
| 4 | Ramp Up | 渐亮 |
| 5 | Ramp Down | 渐灭 |
| 6 | Sine | 正弦脉动 |
| 7 | Random | 随机乱闪 |

> **性能优化**：频闪模式 0/1 时组件自动关闭 Tick，模式 2~7 时自动启用 Tick 进行每帧计算。

### 4.4 效果图案 (Effect Control)

效果组件支持材质驱动的动态图案效果，通过以下参数控制：

| 参数 | 说明 | 范围 | 默认值 |
|------|------|------|--------|
| **Effect** | 效果图案编号/混合程度 | 0.0 ~ 任意 | 0.0 |
| **Speed** | 效果动画播放速度 | 任意 | 1.0 |
| **Width** | 效果图案的宽度/范围 | 0.0 ~ 1.0 | 0.5 |

这些参数通过材质参数 `Effect`、`Speed`、`Width` 控制，具体的视觉效果取决于材质中的实现逻辑（如流水灯、追逐灯、呼吸灯等）。

### 4.5 综合控制 (EffectsControl)

效果组件提供一个「一站式」的综合控制接口，可以一次性设置多个参数：

| 参数 | 说明 |
|------|------|
| **亮度** | 效果面板的自发光亮度 |
| **频闪** | 频闪速度 |
| **颜色** | 发光颜色 |
| **方向** | 效果动画的运行方向 |
| **效果** | 效果图案编号 |
| **速度** | 效果动画速度 |
| **宽度** | 效果图案宽度 |

> **使用建议**：当需要同时更新多个参数时，使用综合控制接口比逐个调用更高效，因为它在一次材质更新中完成所有参数的设置。

### 4.6 效果方向 (Direction)

| 参数 | 说明 | 范围 |
|------|------|------|
| **Direction** | 效果动画的运行方向 | 任意浮点值 |

> **使用场景**：对于流水灯效果，Direction 控制流动的方向（如从左到右、从右到左）。

## 五、使用指南

### 5.1 基本设置流程

1. 在灯具蓝图中添加 SuperEffectComponent
2. 指定效果面板的**静态网格模型**（StaticMeshEffect）
3. 设置**效果面板的位置和大小**（EffectTransform）
4. 选择是否使用**透明材质**（bTransparent）
5. 指定**材质槽索引**（EffectMaterialNo，通常为 0）
6. 在灯具库中映射 DMX 通道到亮度、颜色、频闪、效果等参数

### 5.2 多效果面板

一个灯具可以拥有**多个**效果组件，例如：

```
SuperStageLight
  └── 灯头
        ├── SuperEffectComponent[0]  — 正面 LED 面板
        ├── SuperEffectComponent[1]  — 左侧装饰灯带
        └── SuperEffectComponent[2]  — 右侧装饰灯带
```

每个效果组件都有独立的亮度、颜色和效果控制，可以通过不同的 DMX 通道独立驱动。

### 5.3 效果 + 光照组合

效果组件通常与光照组件**配合使用**：

```
SuperStageLight
  └── 灯头
        ├── SuperBeamComponent   — 主光源（产生光照和光束）
        └── SuperEffectComponent — 效果面板（纯视觉效果）
```

这样灯具既有光照投射能力，又有 LED 面板的视觉效果。

## 六、常见问题

### Q: 效果面板的材质看起来没有效果/全黑？
**A**: 确认以下几点：
1. 已正确指定了 StaticMeshEffect（效果网格模型）
2. EffectMaterialNo 与网格模型的材质槽索引匹配
3. 亮度（Intensity）大于 0
4. 效果面板的可见性未被关闭

### Q: 切换透明/不透明模式后效果异常？
**A**: 系统会在检测到透明状态变化时自动重建动态材质实例。如果仍有异常，可能需要重新调用 SetEffectMaterial 来强制重建。

### Q: 效果组件会产生光照吗？
**A**: 不会。效果组件纯粹是视觉效果（自发光材质），不会照亮周围物体。如果需要让效果面板照亮周围环境，需要另外添加光源组件（如 SuperRectComponent 或 SuperMatrixComponent 的分段光源）。

### Q: 如何让多台灯具的效果面板不同步闪烁？
**A**: 为每台灯具的效果组件设置不同的**随机种子**值（Random Seed）。这样它们在 Random 频闪模式下会产生不同的闪烁时序。

### Q: 效果动画的速度、宽度等参数具体效果是什么？
**A**: 这些参数的具体视觉效果取决于所使用的效果材质（MI_Effect_Inst）中的着色器逻辑。常见的效果包括流水灯、追逐灯、呼吸灯、渐变等。可以通过材质编辑器查看和修改具体的效果实现。
