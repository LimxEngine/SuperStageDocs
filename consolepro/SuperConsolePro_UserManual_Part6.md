# 第一章 DMX 输出

## 1.1 DMX 输出架构

SuperConsolePro 通过底层的 **SuperDMX 模块** 将控制信号发送到灯具。控台内部有多个"值来源"，它们经过优先级合并后输出最终的 DMX 值。

### 值来源与优先级

```
 优先级从高到低：

  ┌──────────────┐
  │  编程器       │  ← 最高优先级（用户正在调整的值）
  │ (Programmer) │
  └──────┬───────┘
         ↓
  ┌──────────────┐
  │  帧效果       │  ← 正在运行的动态效果
  │(Frame Effect)│
  └──────┬───────┘
         ↓
  ┌──────────────┐
  │  CUE 回放    │  ← CUE 面板中正在运行的 CUE
  │   (Cue)      │
  └──────┬───────┘
         ↓
  ┌──────────────┐
  │ 时间码 CUE   │  ← 由外部时间码触发的 CUE
  │(TimecodeCue) │
  └──────┬───────┘
         ↓
  ┌──────────────┐
  │  时间线       │  ← 时间线回放输出
  │ (Timeline)   │
  └──────┬───────┘
         ↓
  ┌──────────────┐
  │  默认值       │  ← 灯具的出厂默认值
  │  (Default)   │
  └──────┬───────┘
         ↓
  ┌──────────────┐
  │  DMX 输出     │  → Art-Net / sACN → 灯具
  │ (SuperDMX)   │
  └──────────────┘
```

### 混合规则

| 通道类型 | 混合方式 | 说明 |
|----------|----------|------|
| **亮度通道**（Dimmer） | HTP（最高值优先） | 多个来源中取最高值 |
| **其他通道**（颜色、位置等） | LTP（最新值优先） | 最后写入的值覆盖之前的值 |

## 1.2 DMX 面板

DMX 面板用于管理 DMX 输出的全局设置。

### 全局 DMX 输出开关

| 开关 | 说明 |
|------|------|
| **DMX Output** | 全局 DMX 输出总开关。**关闭后所有 DMX 输出停止**，但控台内部的编程和回放不受影响 |

> **使用场景**：在编程预演阶段关闭 DMX 输出，避免影响正在使用中的真实灯具。

### Universe 管理

面板列出所有被使用的 DMX Universe：

| 列 | 说明 |
|----|------|
| **Universe 编号** | DMX 域编号（1-64） |
| **使用的通道数** | 该 Universe 中被占用的通道数量 |
| **启用/禁用** | 单独控制该 Universe 是否输出 DMX |

### 单独禁用 Universe

- 每个 Universe 右侧有独立的启用/禁用开关
- 禁用某个 Universe → 该域的 DMX 输出暂停
- 其他 Universe 不受影响

## 1.3 DMX 值显示

### 在灯具表中查看

灯具表面板支持切换值的显示格式：

| 格式 | 显示 | 说明 |
|------|------|------|
| **百分比** | 0% - 100% | 默认显示格式，直观易读 |
| **DMX 值** | 0 - 255 | 原始 DMX 通道值，适合调试 |

切换方式：点击灯具表工具栏中的 **DMX 值显示** 按钮。

### 在编码器中查看

编码器轮上显示的值默认为百分比。对于角度类属性（Pan、Tilt），显示为角度值。

# 第二章 演出文件管理

## 2.1 演出文件概述

演出文件（Show File）是 SuperConsolePro 的项目文件，保存了控台的**全部工作数据**。

### 文件格式

| 项目 | 说明 |
|------|------|
| **扩展名** | `.ssshow` |
| **格式** | 二进制序列化 |
| **存储位置** | `%LOCALAPPDATA%\SuperStage\Shows\` |

### 文件包含的数据

| 数据类别 | 内容 |
|----------|------|
| **配接数据** | 所有灯具的 Fixture ID、名称、类型、Universe/Channel 映射 |
| **灯具组** | 所有灯具组定义（名称、成员、颜色） |
| **CUE 数据** | 所有 CUE List 及其中的每个 CUE（属性值、时间参数、触发模式） |
| **预设数据** | 所有预设（Dimmer / Position / Gobo / Color / Beam / Focus / Control / Shapers） |
| **帧效果预设** | 用户自定义的帧效果预设 |
| **时间线数据** | 所有时间码及其轨道、片段、标记点 |
| **布局数据** | 所有 Layout 及灯具 2D 位置 |
| **窗口布局** | 视图预设（窗口排列方式） |
| **快捷键绑定** | CUE 的键盘快捷键映射 |
| **DMX 设置** | Universe 启用状态等 |

## 2.2 演出文件面板

### 面板布局

```
┌──────────────────────────────────────────────────────────────┐
│ 当前演出: 春晚2026.ssshow                     [未保存更改 *] │
├──────────────────────────────────────────────────────────────┤
│ [新建] [保存] [另存为] [打开]                                 │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ 演出文件列表:                                                 │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ ● 春晚2026.ssshow          2026-01-15 14:30            │ │
│ │   元宵晚会.ssshow          2026-02-05 09:15            │ │
│ │   测试秀.ssshow            2026-01-10 11:00            │ │
│ └──────────────────────────────────────────────────────────┘ │
│                                                              │
│ [删除] [重命名]              ← 计划功能，当前版本暂未实现     │
└──────────────────────────────────────────────────────────────┘
```

## 2.3 新建演出

### 操作步骤

1. 点击 **新建**（New）按钮
2. 弹出新建演出对话框
3. 输入演出名称
4. 点击 **确认**
5. 系统创建空白演出文件

> **注意**：新建演出会清空当前所有数据。如果当前有未保存的更改，系统会先提示保存。

### 新建演出对话框

| 字段 | 说明 |
|------|------|
| **演出名称** | 输入演出名称（不需要输入扩展名） |

## 2.4 保存演出

### 快速保存

- 点击 **保存**（Save）按钮
- 如果已有文件名，直接覆盖保存
- 如果是新建的演出尚未保存过，自动跳转到"另存为"

### 另存为

1. 点击 **另存为**（Save As）按钮
2. 弹出另存为对话框
3. 输入新的文件名
4. 点击 **确认**
5. 文件保存到新路径

### 未保存更改提示

- 标题栏中当前演出名称后面出现 **`*`** 号，表示有未保存的更改
- 关闭控台或新建演出时，系统会提示"是否保存当前更改？"

## 2.5 打开演出

### 从文件列表打开

1. 点击 **打开**（Open）按钮
2. 弹出打开演出对话框
3. 演出文件列表显示所有可用的 `.ssshow` 文件
4. 每个文件显示名称和最后修改时间
5. 选中文件后点击 **确认**

### 自动加载上次演出

- 控台启动时会自动加载上次打开的演出文件
- 如果上次的文件已被删除，则以空白状态启动

## 2.6 删除演出

> **注意**：删除演出功能计划在后续版本中实现。当前版本请通过文件资源管理器手动删除 `%LOCALAPPDATA%\SuperStage\Shows\` 目录下的 `.ssshow` 文件。

## 2.7 重命名演出

> **注意**：重命名演出功能计划在后续版本中实现。当前版本请使用 **另存为** 功能以新名称保存演出文件。

# 第三章 设置面板（Settings）

## 3.1 面板结构

设置面板包含三个标签页：

| 标签页 | 内容 |
|--------|------|
| **Show** | 演出文件管理（新建/保存/另存为/打开） |
| **Patch** | 灯具配接管理 |
| **DMX** | DMX 输出配置 |

## 3.2 Show 标签页

包含与演出文件面板相同的功能（新建、保存、另存为、打开）。这是一个便捷入口，无需单独打开演出文件面板。

详细操作请参考本手册第二章"演出文件管理"。

## 3.3 Patch 标签页

包含与配接面板相同的功能（扫描场景、导入/移除灯具、同步）。这是一个便捷入口。

详细操作请参考用户手册（二）第一章"灯具配接"。

## 3.4 DMX 标签页

### DMX 输出总开关

| 设置项 | 说明 |
|--------|------|
| **Enable DMX Output** | 全局 DMX 输出开关。关闭后所有 DMX 信号停止发送 |

### Universe 配置

列出所有使用中的 DMX Universe：

| 设置项 | 说明 |
|--------|------|
| **Universe 编号** | 域编号（只读，由灯具配接决定） |
| **启用** | 是否输出该域的 DMX 信号 |
| **协议** | DMX 输出协议（由 SuperDMX 模块配置决定） |

# 第四章 快捷键绑定

## 4.1 CUE 快捷键

可以为 CUE 绑定键盘快捷键，实现按键触发 CUE。

### 绑定快捷键

1. 在回放面板中**右键点击** CUE 执行器按钮
2. 选择 **Bind Key**（绑定快捷键）
3. 在弹出对话框中按下想要绑定的按键组合
4. 支持以下修饰键组合：
   - 单独按键（如 `F1`、`1`、`A`）
   - **Ctrl** + 按键
   - **Shift** + 按键
   - **Alt** + 按键
   - 多修饰键组合（如 **Ctrl + Shift + F1**）
5. 系统自动检测冲突 — 如果该快捷键已被其他 CUE 使用，会提示是否覆盖

### 解除绑定

1. **右键点击** CUE 执行器按钮
2. 选择 **Unbind Key**（解除绑定）

### 快捷键行为

| 触发模式 | 按下按键 | 松开按键 |
|----------|----------|----------|
| **Toggle** | 激活/释放 CUE（切换） | 无动作 |
| **Momentary** | 激活 CUE | 释放 CUE |
| **Flash** | 瞬间激活 CUE（无渐变） | 瞬间释放 CUE |

### 查看已绑定的快捷键

- 已绑定快捷键的 CUE 执行器按钮上会显示快捷键标记
- 在 CUE 的右键菜单中可以看到当前绑定的快捷键

# 第六章 常见工作流程

## 6.1 从零开始的完整演出编程流程

```
第 1 步：准备
├── 在场景中摆放 SuperDMX 灯具
├── 打开 SuperConsolePro
└── 新建演出文件

第 2 步：配接
├── 设置面板 → Patch → 扫描场景
├── 导入所有灯具
└── 确认 Fixture ID 分配

第 3 步：分组
├── 按区域选择灯具
├── 保存为灯具组（如"面光"、"侧光"、"染色"、"效果"）
└── 为灯具组设置不同颜色便于识别

第 4 步：建立预设库
├── 逐组选择灯具
├── 调整常用位置 → 存储为 Position 预设
├── 调整常用颜色 → 存储为 Color 预设
└── 调整常用光束 → 存储为 Beam 预设

第 5 步：编程 CUE
├── 选择灯具 → 调整属性 → Store CUE
├── 为每个 CUE 设置合适的渐变时间
├── 在回放面板中测试每个 CUE
└── 重复直到所有场景完成

第 6 步：编排时间线（可选）
├── 创建时间码
├── 导入背景音乐
├── 将 CUE 放置到时间轴
├── 添加标记点
└── 预览播放，微调时间

第 7 步：保存
└── 保存演出文件
```

## 6.2 现场演出工作流

```
演出前：
├── 打开演出文件
├── 确认 DMX 输出已开启
├── 检查所有灯具是否在线
└── 测试关键 CUE

演出中（手动模式）：
├── 按顺序点击 Go 执行 CUE
├── 使用快捷键快速触发效果
├── 必要时手动调整编程器值
└── 使用 Release All 紧急全灭

演出中（时间线模式）：
├── 点击 Play 开始时间线
├── 时间线自动触发 CUE
├── 必要时暂停/跳转
└── 手动叠加编程器值做微调

演出后：
└── 保存演出文件（如有修改）
```

## 6.3 帧效果快速创建流程

```
1. 选择灯具（建议选择一排或一组灯具）
2. 打开帧编辑器
3. 选择波形类型（追光用 Sine，频闪用 Rectangle）
4. 设置速度（BPM）
5. 调整相位范围（360° = 完整追光）
6. 设置 Wings（镜像效果）
7. 点击 Play 预览
8. 满意后存储为 CUE（包含帧效果）
```

## 6.4 颜色追光效果创建

```
1. 选择一排灯具（注意选择顺序从左到右）
2. 打开帧编辑器
3. 添加 3 个步骤：
   - Step 1: Red=100%, Green=0%, Blue=0%
   - Step 2: Red=0%, Green=100%, Blue=0%
   - Step 3: Red=0%, Green=0%, Blue=100%
4. 波形类型：Sine（平滑过渡）
5. 速度：30 BPM（较慢，优雅的颜色变化）
6. 相位范围：360°
7. 播放预览 → 灯具呈现彩虹追光效果
```

# 附录 B：故障排除

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 灯具不响应控台操作 | DMX 输出已关闭 | 检查 DMX 面板 → 开启 DMX Output |
| 灯具不响应控台操作 | 灯具未配接 | 设置面板 → Patch → 扫描并导入灯具 |
| 灯具不响应控台操作 | 灯具未选中 | 确认灯具表中灯具已被选中（蓝色高亮） |
| 编码器调整没效果 | 编码器在时间模式 | 检查编码器模式 → 切换回 Value |
| CUE 不输出 | 编程器覆盖了 CUE 值 | 点击 Clear 清除编程器 |
| CUE 渐变很慢 | Fade In 时间设置过长 | 编辑 CUE → 修改 Fade In 时间 |
| 帧效果不运行 | 未点击 Play | 帧编辑器 → 点击 ▶ Play 按钮 |
| 帧效果所有灯具同步 | 相位范围为 0° | 设置 Phase Range 为 360° |
| 时间线播放无反应 | 没有 CUE 片段 | 在 CUE 轨道上添加 CUE 片段 |
| 音频不播放 | 音频轨道被静音 | 检查轨道的 [M] 按钮 |
| 演出文件打不开 | 文件损坏或版本不兼容 | 尝试从备份恢复 |
| 窗口布局混乱 | 视图预设被覆盖 | 使用 Clear Screen 重置，重新排列 |
