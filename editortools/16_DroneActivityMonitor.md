# SuperStage 无人机活动监控 用户手册

## 1. 概述

无人机活动监控器（LDLink Activity Monitor）用于实时查看通过 LDLink 协议连接的无人机灯光设备的状态信息。面板提供接收器开关、网络配置、实时统计和无人机数据列表。

---

## 2. 打开方式

**打开方式**：编辑器底部状态栏 → 点击 **LDLink** 按钮

---

## 4. 接收器配置

### 4.1 Enable Receiver 复选框

| 状态 | 操作 |
|------|------|
| 勾选 | 启动 LDLink 数据接收器，开始接收无人机状态数据 |
| 取消 | 停止接收器 |

启动时会自动应用当前的 Bind IP 和 Port 配置到 `UDroneLinkSubsystem`。

### 4.2 网络参数

| 参数 | 控件 | 说明 | 默认值 |
|------|------|------|--------|
| **Bind IP** | 下拉列表 | 绑定的本地网络接口（自动枚举所有可用网卡） | 0.0.0.0（所有接口） |
| **Port** | 数值输入框 | 监听端口号（范围 1 – 65535） | 从 `UDroneLinkSubsystem` 读取 |

### 4.3 Clear 按钮

点击 **"Clear"** 清除当前所有无人机数据。

---

## 5. 实时统计

统计栏显示接收器的运行状态，格式为单行文本：

`Drones: {N} | Packets: {N}/s | Bytes: {N}/s | Errors: {N}`

| 统计项 | 说明 |
|--------|------|
| **Drones** | 当前连接的无人机总数（超出显示上限时额外提示 `showing {M}`） |
| **Packets** | 每秒接收到的数据包数量 |
| **Bytes** | 每秒接收到的字节数 |
| **Errors** | 累计的数据解析错误数 |

---

## 6. 无人机列表

### 6.1 列表字段

| 列名 | 宽度 | 说明 |
|------|------|------|
| **ID** | 40px | 无人机的唯一编号 |
| **Status** | 60px | 无人机当前状态 |
| **Position** | 180px | 无人机 3D 位置坐标（X, Y, Z） |
| **Yaw** | 60px | 无人机偏航角 |
| **LED** | 40px | 当前 LED 灯光颜色预览 |

### 6.2 性能限制

为控制 Slate UI 性能，当无人机数量超过内部上限（`MaxDronesToShow`）时，列表仅显示前 N 架无人机，统计栏会提示实际总数。

---

## 7. 数据刷新

- 面板每 **200ms** 刷新一次无人机数据快照
- 仅当无人机集合发生变化（加入/离开）时才重建列表 UI
- 数值类字段（Position、Yaw、LED）通过 Lambda 绑定实时更新，无需重建行

---

## 8. 使用场景

### 场景 1：演出前检查

1. 打开无人机活动监控器
2. 勾选 **Enable Receiver** 启动接收
3. 确认所有无人机出现在列表中
4. 检查 Status、Position 是否正常

### 场景 2：演出实时监控

1. 监控飞行中的无人机状态
2. 观察 LED 颜色是否与预编排一致
3. 关注 Errors 计数是否异常

### 场景 3：排查连接问题

1. 检查 Drones 计数是否为 0
2. 确认 Packets/s 是否有数据
3. 尝试更换 Bind IP 到具体网卡地址

---

## 9. 注意事项

- 此面板为只读监控，不能通过此面板控制无人机
- 需要勾选 Enable Receiver 才能开始接收数据
- Bind IP 下拉列表会自动枚举本机所有网络接口
- 关闭面板不会停止 `UDroneLinkSubsystem` 的接收器
- 面板刷新频率为 200ms（5Hz），不等同于协议接收频率

---

## 10. 常见问题

| 问题 | 解决方法 |
|------|----------|
| 列表中没有无人机 | 确认 Enable Receiver 已勾选，且端口设置正确 |
| Packets/s 为 0 | 检查 Bind IP 和 Port 是否与无人机发送端匹配 |
| Errors 持续增长 | 可能存在数据包格式不兼容，检查无人机固件版本 |
| 只显示部分无人机 | 超出 MaxDronesToShow 上限，统计栏会提示实际总数 |
| 数据更新缓慢 | 面板以 200ms 间隔刷新，属正常行为 |
