# SuperStage DMX 活动监控 用户手册

## 1. 概述

DMX 活动监控器（DMX Activity Monitor）是一个实时 DMX 数据查看工具，以灰度柱状条方式直观显示 Universe 通道电平。支持监视单个 Universe 或自动扫描所有活跃 Universe，帮助你快速验证 DMX 信号的接收和发送情况。

---

## 2. 打开方式

DMX 活动监控器可嵌入在 **DMX 配置面板** 中，或作为独立控件使用。

**主菜单路径**：工具栏 **SuperStage** 下拉菜单 → **SuperDMXTool** → **DMXConfiguration**

## 4. 功能说明

### 4.1 监控模式

| 控件 | 说明 |
|------|------|
| **All Universes 复选框** | 勾选：自动显示所有有信号的 Universe；不勾选：仅显示指定 Universe |
| **Universe 输入框** | 选择要监视的 Universe 编号（范围 1 – 32767），支持拖拽调整 |

### 4.2 通道显示

每个 Universe 显示为一行，包含：
- **左侧标签**：`U {编号}`
- **通道列**：每个通道占 28 像素宽，从上到下依次显示：
  - **通道地址**（1–512）
  - **灰度柱状条**（20 像素高）：颜色亮度 = 通道值 / 255，值越高越亮
  - **数值**（0–255）

可显示的通道数量可配置（默认 512，最大 512）。

### 4.3 信号过滤

- **All Universes 模式**：从 `USuperDMXSubsystem` 获取所有已知 Universe，只显示包含非零信号的 Universe
- **单 Universe 模式**：仅查询指定 Universe 的缓冲区

当没有 DMX 数据时，显示提示文字：*"No DMX data yet. Check Input Enable and source."*

### 4.4 Clear 按钮

点击 **"Clear"** 按钮清除 DMX 缓冲区：
- **All Universes 模式**：清除所有 Universe 的缓冲区
- **单 Universe 模式**：仅清除当前选中的 Universe

## 5. 使用场景

### 场景 1：验证控台信号是否到达

1. 在 DMX 配置面板中设置好输入
2. 打开 DMX 活动监控器
3. 勾选 All Universes 或选择对应 Universe
4. 在控台上推动推杆
5. 观察对应通道的灰度条和数值是否变化

### 场景 2：检查灯具 Patch 是否正确

1. 选择灯具所在的 Universe
2. 观察灯具起始地址开始的几个通道
3. 对比通道数量是否与灯库定义的通道跨度一致

### 场景 3：排查 DMX 信号问题

1. 如果灯具不响应，先在监控器中检查对应通道是否有数据
2. 如果监控器无数据 → 检查网络连接和 DMX 输入设置
3. 如果监控器有数据但灯具不响应 → 检查灯具的 Patch 设置

---

## 6. 注意事项

- 活动监控器每 **100ms** 刷新一次数据
- 仅当 Universe 集合发生变化时才重建 UI（避免频繁重绘）
- 监控器是只读的，不能通过此面板修改 DMX 值（Clear 除外）
- 需要有效订阅权限才能刷新数据（无权限时数据保持不变，不会闪烁）
- Universe 编号范围为 1–32767（与 SuperDMX 子系统一致）
- 支持水平和垂直双向滚动，适应大量通道的查看需求
