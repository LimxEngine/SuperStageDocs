# SuperStage 灯具阵列工具 用户手册

## 1. 概述

灯具阵列工具（Fixture Array Tool）是 SuperStage 的交互式灯具批量排列工具。它可以将一个已有的灯具按照 **线性**、**网格** 或 **环形** 模式复制成阵列，并提供实时预览功能。在确认排列效果后一键创建所有灯具，极大提高了大规模灯具布局的效率。

---

## 2. 打开方式

**打开方式**：在编辑器左侧模式选择栏中切换到 **SuperStage 编辑模式**，然后在模式工具栏中选择 **灯具阵列工具**。

> **前提条件**：使用前必须先在 3D 视口中选中一个 SuperStage DMX 灯具作为"源灯具"。

---

## 3. 使用流程

### 步骤 1：选择源灯具

在 3D 视口中选中一个灯具 Actor。这个灯具将作为阵列的模板——阵列中的每个新灯具都会复制它的类型和配置。

### 步骤 2：打开阵列工具

通过主菜单打开灯具阵列工具。工具会自动检测你选中的灯具。

### 步骤 3：选择阵列模式

在工具面板中选择阵列排列模式（线性 / 网格 / 环形）。

### 步骤 4：调整参数

根据选择的模式，调整相应的参数。场景中会实时显示半透明的 **预览灯具**，让你在创建前就能看到排列效果。

### 步骤 5：创建阵列

确认预览效果满意后，点击 **"创建"** 按钮。所有预览灯具将转为正式的场景灯具。

> **提示**：源灯具在阵列创建完成后会被移除，其位置由阵列中的第一个灯具取代。

---

## 4. 阵列模式详解

### 4.1 线性阵列 (Linear)

将灯具沿一条直线等间距排列。

```
[灯1] ——间距—— [灯2] ——间距—— [灯3] ——间距—— [灯4]
```

#### 参数

| 参数 | 类型 | 说明 | 范围 | 默认值 |
|------|------|------|------|--------|
| **Count** | int32 | 阵列中的灯具数量 | 1 - 100（UI 上限 50） | 5 |
| **Spacing** | FVector | 相邻灯具之间的三维间距向量（厘米） | 无限制 | (100, 0, 0) |

> **注意**：Spacing 是一个三维向量，可以同时控制 X/Y/Z 三个方向的间距，实现斜向排列。

#### 示例

- **一排 Truss 灯**：Count=8，Spacing=(150, 0, 0)
- **斜向排列**：Count=5，Spacing=(100, 0, 50)

---

### 4.2 网格阵列 (Grid)

将灯具按二维网格排列（行列矩阵）。

```
[灯1] [灯2] [灯3] [灯4]
[灯5] [灯6] [灯7] [灯8]
[灯9] [灯10][灯11][灯12]
```

#### 参数

| 参数 | 类型 | 说明 | 范围 | 默认值 |
|------|------|------|------|--------|
| **Count X** | int32 | 水平方向的灯具数量 | 1 - 50（UI 上限 20） | 3 |
| **Count Y** | int32 | 垂直方向的灯具数量 | 1 - 50（UI 上限 20） | 3 |
| **Spacing X** | float | 水平方向相邻灯具的间距（厘米） | ≥ 1.0 | 100.0 |
| **Spacing Y** | float | 垂直方向相邻灯具的间距（厘米） | ≥ 1.0 | 100.0 |
| **Plane** | EFixtureArrayAxis | 网格所在平面的法线方向 | X / Y / Z | Z |
| **Center At Source** | bool | 是否以源灯具位置为网格中心 | 开/关 | 开 |
| **Honeycomb Offset** | float | 奇偶行水平偏移量（0 = 无偏移，0.5 = 半间距蜂窝排列） | 0.0 - 1.0（UI 上限 0.5） | 0.0 |

#### 平面法线轴说明

| 法线轴 | 网格平面 | 适用场景 |
|--------|----------|----------|
| **Z** | 水平面（XY 平面） | 地面灯阵、天花板灯阵 |
| **Y** | 前后立面（XZ 平面） | 背景墙灯阵 |
| **X** | 左右立面（YZ 平面） | 侧面灯阵 |

#### 蜂窝偏移说明

Honeycomb Offset 为连续值（非开关），设为 0.5 时效果：
```
普通网格 (0.0):       蜂窝网格 (0.5):
[1] [2] [3] [4]       [1] [2] [3] [4]
[5] [6] [7] [8]         [5] [6] [7] [8]
[9] [10][11][12]      [9] [10][11][12]
```

奇数行（第 2、4 行...）会向右偏移 `SpacingX × HoneycombOffset`，值为 0.5 时形成标准蜂窝状/砖墙状排列。

---

### 4.3 环形阵列 (Radial)

将灯具沿圆形或圆弧排列。

```
        [灯2]
    [灯1]     [灯3]
              
  [灯8]         [灯4]
              
    [灯7]     [灯5]
        [灯6]
```

#### 参数

| 参数 | 类型 | 说明 | 范围 | 默认值 |
|------|------|------|------|--------|
| **Count** | int32 | 圆周上的灯具数量 | 2 - 100（UI 上限 36） | 8 |
| **Radius** | float | 圆的半径（厘米） | ≥ 10.0（UI 起始 50.0） | 200.0 |
| **Start Angle** | float | 第一个灯具的角度位置（度） | 0° - 360° | 0° |
| **End Angle** | float | 最后一个灯具的角度位置（度，360 = 完整圆） | 0° - 360° | 360° |
| **Rotation Axis** | EFixtureArrayAxis | 环形阵列的旋转轴 | X / Y / Z | Z |
| **Orient To Center** | bool | 每个灯具是否自动旋转朝向圆心 | 开/关 | **开** |
| **Include Center** | bool | 是否在圆心位置额外放置一个灯具 | 开/关 | 关 |

#### 角度说明

- **起始角度 0° / 结束角度 360°**：完整圆环（首尾不重叠）
- **起始角度 0° / 结束角度 180°**：半圆弧
- **起始角度 45° / 结束角度 315°**：270° 的圆弧

#### 朝向中心

| 状态 | 说明 |
|------|------|
| **关** | 所有灯具保持源灯具的原始朝向 |
| **开** | 每个灯具自动旋转，使其正面朝向圆心 |

适合环形灯阵需要所有灯都照向中心舞台的场景。

#### 包含中心

| 状态 | 说明 |
|------|------|
| **关** | 仅在圆周上放置灯具 |
| **开** | 圆周上放置灯具 + 圆心位置额外放置 1 个灯具 |

---

## 5. 通用参数

以下参数对所有阵列模式生效，位于 **Common** 分类下：

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| **Array Rotation** | FRotator | 整个阵列绕源灯具位置的旋转（Pitch/Yaw/Roll） | (0, 0, 0) |
| **Item Rotation** | FRotator | 每个灯具的额外旋转偏移（Pitch/Yaw/Roll） | (0, 0, 0) |

---

## 6. 状态信息

工具面板顶部的 **Status** 分类显示以下只读信息：

| 字段 | 说明 |
|------|------|
| **Status** | 当前状态提示（如 "Select a Fixture Actor"） |
| **Source Fixture** | 源灯具名称 |
| **Preview Count** | 当前预览灯具的数量 |

---

## 7. 实时预览

调整任何参数时，场景中会立即更新预览灯具的位置和旋转。预览灯具的特征：
- 使用 `RF_Transient` 标记，不会保存到场景
- 不会发送/接收 DMX 信号
- 参数变化会标记 `bPreviewDirty`，在下一帧自动刷新

---

## 8. 操作按钮与撤销

### 操作按钮

| 按钮 | 说明 |
|------|------|
| **Create Fixtures** | 销毁预览 Actor，销毁源灯具，然后关闭工具（创建操作在 Transaction 中完成，支持撤销） |
| **Done** | 完成并退出工具 |

### 撤销

创建操作支持 **Ctrl+Z 撤销**。

---

## 9. 注意事项

- 必须先选中一个有效的 SuperStage DMX 灯具才能使用此工具
- 如果选中了多个灯具，工具只使用第一个选中的灯具作为源
- 非 SuperStage DMX 灯具（如普通点光源、聚光灯）无法作为源灯具
- 创建大量灯具（如 100+）可能会影响编辑器性能
- 阵列中所有灯具使用与源灯具相同的灯具类型
- 建议创建阵列后使用批量 Patch 工具重新分配 DMX 地址

---

## 10. 常见问题

| 问题 | 解决方法 |
|------|----------|
| 打开工具后无预览 | 确认已在视口中选中了一个 SuperStage 灯具 |
| 灯具排列方向不对 | 调整"方向"或"平面法线轴"参数 |
| 间距看起来太大/太小 | 注意间距单位是厘米，1 米 = 100 厘米 |
| 创建后灯具 DMX 地址冲突 | 使用批量 Patch 工具重新分配地址 |
| 环形阵列首尾重叠 | 检查结束角度是否设为 360°，是的话最后一个灯具不会与第一个重叠 |
